# 仿真系统设计文档

## 1. 系统总览

本仿真系统用于模拟新能源物流车队的动态调度。系统包含以下核心模块：

- **车队模型**：多车型、异构性质
- **货物系统**：K类货物分类、类型约束
- **能量管理**：基于速度的耗电计算
- **充电站仿真**：FIFO队列、实时充电管理
- **路网模型**：Dijkstra/A* 路径规划、动态拥堵
- **任务生成**：随机任务分布、时间窗约束

## 2. 车队模型 (Vehicle System)

### 2.1 车型分类

定义3种标准车型，通过 `VehicleType` 模板管理：

| 车型 | 电池(kWh) | 载重(kg) | 体积(m³) | 最速(km/h) | 充电(kWh/h) | 支持货物 |
|------|----------|---------|---------|----------|-----------|--------|
| Compact | 60 | 500 | 2.5 | 80 | 30 | type_1, type_2 |
| Standard | 100 | 1000 | 5.0 | 100 | 50 | type_1, type_3 |
| Large | 150 | 2000 | 10.0 | 90 | 75 | type_1, type_4 |

### 2.2 车辆状态 (VehicleStatus)

```python
class VehicleStatus(Enum):

    MAINTENANCE = "maintenance"  # 维修中
```

- `min_battery_threshold`: 最低电量阈值 (kWh，默认20)

#### 3.2.2 载重管理

## 4. 能量管理系统 (Energy System)

### 4.1 能耗计算模型

**能耗公式**：
```python
def calculate_consumption(distance, load, speed_kmh=40.0, efficiency=0.15, weather_factor=1.0):
    # 基础耗能
    load_factor = 1.0 + load / 10000  # 每10kg增加0.1%
    base = distance * efficiency * load_factor * weather_factor
    
    # 速度因子：40km/h最优
    speed_factor = 1 + ((speed_kmh - 40)² / 3200)
    
    # 最终耗能
    energy = base * speed_factor
    return energy
```

### 4.2 耗能影响因素

| 因素 | 说明 | 数值范围 |
|------|------|---------|
| **距离** | 行驶距离 (km) | 连续 |
| **货重** | 货物重量 (kg) | 线性增加耗能 |
| **速度** | 驾行速度 (km/h) | 40km/h最优（100%），偏离增加耗能 |
| **效率** | 车型固有效率 (kWh/km) | 0.12-0.20 |
| **天气** | 天气倍率 | 1.0 (默认) |

### 4.3 速度对耗能的影响

```
速度     因子    耗能
20 km/h  1.1   基础×110%
40 km/h  1.0   基础×100% ← 最优
60 km/h  1.1   基础×110%
80 km/h  1.2   基础×120%
```

## 5. 充电站系统 (Charging Station System)

### 5.1 充电站配置

| 属性 | 说明 | 默认值 |
|------|------|--------|
| `id` | 充电站ID | - |
| `position` | 地理位置 | - |
| `num_chargers` | 充电器数量（容量） | 2 |
| `charging_power` | 充电功率 (kWh/h) | 50 |

### 5.2 排队规则 (FIFO)

```
新车到达
   ↓
enqueue() → waiting_queue
   ↓
有空充电器 → dequeue()
   ↓
start_charging() → charging_vehicles
   ↓
充电完成 → complete_charging()
```

### 5.3 充电记录

```python
@dataclass
class ChargingRecord:
    vehicle_id: str              # 车辆ID
    start_time: datetime         # 开始时间
    duration_minutes: float      # 充电耗时（分钟）
    target_energy: float         # 目标电量 (kWh)
    end_time: Optional[datetime] # 结束时间
```

### 5.4 充电站方法

| 方法 | 功能 |
|------|------|
| `enqueue(vehicle_id)` | 加入排队，返回位置 |
| `dequeue()` | 从排队出队 |
| `get_queue_position(vehicle_id)` | 查询队位 |
| `start_charging(vehicle_id, target_energy, start_time)` | 开始充电 |
| `complete_charging(vehicle_id, end_time)` | 完成充电 |
| `get_wait_time()` | 获得估计等待时间 (分钟) |
| `get_queue_length()` | 获得队列总长 |
| `get_status()` | 获得状态快照 |

### 5.5 充电时间计算

```python
duration_hours = energy_needed / charging_power  # 充电时间 (小时)
duration_minutes = duration_hours * 60           # 转换为分钟
```

## 6. 路网模型 (Road Network)

### 6.1 路网结构

- 基于NetworkX图论库实现
- 支持Dijkstra和A*两种路径规划算法
- 包含动态拥堵模型

### 6.2 关键方法

| 方法 | 功能 |
|------|------|
| `dijkstra(start, end)` | 计算最短距离 |
| `a_star(start, end)` | A*算法最短距离 |
| `shortest_distance(start, end, method)` | 统一接口 |
| `shortest_travel_time_hours(start, end, start_time)` | 计算行驶时间（考虑拥堵） |

### 6.3 拥堵模型 (CongestionModel)

```python
class CongestionModel:
    morning_peak: Tuple[int, int] = (7, 10)      # 早高峰
    evening_peak: Tuple[int, int] = (17, 20)     # 晚高峰
```

## 7. 仿真引擎 (Simulator)

### 7.1 初始化参数

```python
Simulator(
    network: RoadNetwork,
    vehicles: List[Vehicle],
    charging_stations: List[ChargingStation],
    dispatcher: Dispatcher,
    cargo_config: CargoConfig = None,
)
```

### 7.2 货物生成

仿真中随机生成任务，遵循货物分布规则：

```python
def generate_random_task(current_time: datetime) -> Task:
    # 按CargoConfig分布生成货物类型
    # TYPE_1: 70%, TYPE_2: 10%, TYPE_3: 10%, TYPE_4: 10%
    
    # 随机参数
    - origin: 随机起点
    - destination: 随机终点
    - weight: 10-500 kg
    - volume: 0.1-3.0 m³
    - deadline: 当前时间后1-8小时
```

### 7.3 运行流程

```
for each step in num_steps:
  1. 生成新任务 (tasks_per_step 个)
  2. 调度器分配任务给车辆
  3. 检查:
     - 货物类型是否匹配
     - 车辆是否有足够电量
     - 车辆是否有足够容量
  4. 执行任务/失败
  5. 时间推进 (1小时)
```

### 7.4 任务执行检查

```python
# 任务失败的三个条件：
1. 货物类型不匹配 → score -= 50
2. 电量不足 → score -= 50
3. 容量不足 → score -= 50

# 任务成功：
score += 100 - (distance × 0.1) - (time_hours × 2.0)
```

### 7.5 仿真输出指标

```python
{
    "completed": int,              # 完成任务数
    "failed": int,                 # 失败任务数
    "total_distance": float,       # 总行驶距离 (km)
    "total_score": float,          # 总评分
    "avg_score_per_task": float,   # 平均每任务评分
    "charging_events": int,        # 充电次数
}
```

## 10. 使用示例

### 10.1 创建仿真场景

```python
from ev_dispatch.scenarios.default import build_default_scenario, CargoConfig
from ev_dispatch.simulator.simulator import Simulator
from ev_dispatch.algorithms.strategies import DispatcherNearestFirst

# 构建场景
network, vehicles, stations = build_default_scenario(
    vehicle_mix={"compact": 2, "standard": 5, "large": 1}
)

# 设置货物配置
cargo_config = CargoConfig(type_1_ratio=0.8)

# 创建仿真
simulator = Simulator(
    network=network,
    vehicles=vehicles,
    charging_stations=stations,
    dispatcher=DispatcherNearestFirst(network),
    cargo_config=cargo_config,
)

# 运行仿真
results = simulator.run_simulation(num_steps=100, tasks_per_step=5)

print(f"完成任务: {results['completed']}")
print(f"失败任务: {results['failed']}")
print(f"总分数: {results['total_score']}")
```
