### 🧑‍💻 成员 A（算法负责人）：调度策略与路径规划优化

**核心职责：** 负责调度算法的上下限，实现动态调度大模型的接入，求解大作业要求的“全局最优解”和“进阶启发式/强化学习算法”。
**对应修改模块：** algorithms 与 network.py

*   **基础策略完善**：在 `algorithms/strategies.py` 现有的“最近任务优先”和“最大任务优先”基础上，增加更合理的基准策略（如考虑充电站距离和排队的综合启发式调度）。
*   **进阶算法探索 (加分核心)**：引入高阶解法，例如使用遗传算法、模拟退火等元启发式算法，或强化学习处理实时派单任务分配（在 `algorithms/` 中新增对应代码实现）。
*   **多车协同分配**：在算法层面上实现“多车协作完成同一超重任务”的切分与分配逻辑。
*   **静态全局最优对比 (上帝视角)**：跳出仿真循环，使用 Gurobi 或 CPLEX 写一个离线的整数规划模型脚本。读取特定场景已生成的所有已知任务，求解理论最高分（全局最优解），最后在结果报告里与动态策略作比对。
*   **路网扩展优化**：完善 `core/network.py` 中的寻路算法（如针对多车请求优化 A* 启发因子），让图结构的生成规则更贴近真实城市路网。

**核心职责：** 搭建出贴近真实的动态物理环境与评价体系，确保所有的作业约束条件都被严丝合缝地触发、记录和计算。
**对应修改模块：** simulator、scenarios 与 core

*   **场景规模扩容**：在 `scenarios/` 目录下（如补充 `medium.py`, `large.py` 脚本），构造至少**三种以上不同规模**的问题（节点数量、车辆数、任务生成频率梯次增加）。
